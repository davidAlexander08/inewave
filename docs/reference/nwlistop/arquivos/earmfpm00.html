Energia Armazenada Final Percentual por Submercado (earmfpm00x.out)¶

Visão geral do modelo¶

Os valores de energias armazenadas finais percentuais por submercado, localizados nos arquivos geralmente denominados earmfpm00x.out, onde x varia confomrme o submercado, são armazenados na classe:

class Earmfpm00(mes_pmo: int, ano_pmo: int, versao_newave: str, submercado: str, energias_armazenadas: Dict[int, numpy.ndarray])[código fonte]¶

Armazena os dados das saídas referentes às energias armazenadas finais, por submercado e em % da energia armazenável máxima.

Esta classe lida com as informações de saída fornecidas pelo NWLISTOP e reproduzidas nos earmfpm00x.out, onde x varia conforme o submercado em questão.

Parâmetros

  • mes_pmo: int

  • ano_pmo: int

  • versao_newave: str

  • submercado: str

  • energias_armazenadas: Dict[int, np.ndarray]

property energias_por_ano¶

Energias armazenadas para cada ano e em cada cenário, para todos os meses, organizadas primeiramente por ano.

Retorna

Dict[int, np.ndarray]

Sobre

O acesso é feito com [ano] e retorna um np.ndarray com os valores de EARM para todos os cenários e meses, naquele ano.

property energias_por_ano_e_cenario¶

Energias armazenadas para cada ano e cenário, para todos os meses, organizadas primeiramente por ano.

Retorna

Dict[int, Dict[int, np.ndarray]]

Sobre

O acesso é feito com [ano][cenario] e retorna um np.ndarray com os valores de EARM para todos os meses, naquele ano e cenário.

property energias_por_ano_e_mes¶

Energias armazenadas para cada ano e mês em cada cenário, para todos os meses, organizadas primeiramente por ano.

Retorna

Dict[int, Dict[int, np.ndarray]]

Sobre

O acesso é feito com [ano][mes] e retorna um np.ndarray com os valores de EARM para todos os cenários, naquele ano e mês.

Leitura¶

A leitura dos arquivos earmfpm00x.out é feita através da classe LeituraEarmfpm00.

class LeituraEarmfpm00(diretorio: str)[código fonte]¶

Realiza a leitura dos arquivos earmfpm00x.out existentes em um diretório de saídas do NEWAVE.

Esta classe contém o conjunto de utilidades para ler e interpretar os campos de arquivos earmfpm00x.out, construindo objetos Earmfpm00 cujas informações são as mesmas dos arquivos.

Este objeto existe para retirar do modelo de dados a complexidade de iterar pelas linhas do arquivo, recortar colunas, converter tipos de dados, dentre outras tarefas necessárias para a leitura.

Uma vez realizada a leitura do arquivo, as informações são guardadas internamente no atributo earmfpms.

Exemplos

>>> diretorio = "~/documentos/.../deck"
>>> leitor = LeituraEarmfpm00(diretorio)
>>> leitor.le_arquivos()
# Ops, esqueci de pegar o objeto
>>> earm_sudeste = leitor.earmfpms['SUDESTE']
le_arquivos() → Dict[str, inewave.nwlistop.modelos.earmfpm00.Earmfpm00][código fonte]¶

Lê os arquivos earmfpm00x.out em um diretório.
